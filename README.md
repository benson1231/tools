## Repository Structure and Design Philosophy

This repository adopts a **decoupled architecture** that clearly separates **computation**, **artifacts**, and **presentation**.

* **`scripts/`**
  Contains all executable source code (e.g., Bash, R, and Python scripts). These scripts are intended to be run independently of the website build process and are responsible for generating analysis results, figures, and reports.

* **`website/assets/`**
  Serves as a read-only artifact store for the website. Outputs generated by scriptsâ€”such as rendered HTML reports, figures, and tablesâ€”are written here and consumed by the website for visualization.

* **`website/docs/`**
  Contains Quarto (`.qmd`) files used for documentation, tutorials, and methodological explanations. These files do not execute analysis code; instead, they describe workflows and reference pre-generated outputs from `website/assets/`.

With this design, **all computation is fully decoupled from presentation**.
Quarto and GitHub Actions are used exclusively for rendering documentation and visualizing existing results, ensuring stable CI builds, reproducibility, and long-term maintainability.

## Clone This Repository

To run the tools locally, clone this repository:

```bash
git clone https://github.com/benson1231/tools.git
cd tools
```

Please refer to the environment documentation in [`envs/README.md`](./envs/README.md) to set up a runnable environment.

---

## Render the Website

To build the website locally, install [Quarto](https://quarto.org/docs/get-started/) or create a ready-to-use environment using [`envs/quarto.yaml`](./envs/quarto.yaml).

```bash
quarto render ./website
```
